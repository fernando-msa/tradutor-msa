# Contribuindo

Obrigado por contribuir com o Tradutor MSA.

## Fluxo recomendado
1. Abra uma issue descrevendo bug/melhoria.
2. Crie uma branch com prefixo semântico (`feat/`, `fix/`, `docs/`, `chore/`).
3. Faça alterações pequenas e auditáveis.
4. Atualize documentação e changelog quando aplicável.
5. Abra PR usando o template do repositório.

## Diretrizes técnicas
- Manter compatibilidade Manifest V3.
- Evitar dependências desnecessárias.
- Priorizar segurança, clareza e baixo acoplamento.
- Não usar `innerHTML` para conteúdo vindo de usuário/API.

## Qualidade
- Teste manual mínimo: fluxo de tradução, histórico, copiar e menu de contexto.
- Para mudanças de UI, anexar screenshot no PR quando possível.
